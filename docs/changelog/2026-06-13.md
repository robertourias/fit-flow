### Bug fix — proxy ECONNREFUSED

- `NEXT_PUBLIC_API_URL=http://localhost:3001` (browser-facing) era reaproveitado por código server-side dentro do container `web`, resolvendo para o próprio container em vez do container `api`.
- Novo env `API_INTERNAL_URL` (default `http://api:3001`, Docker-internal), usado em `apiFetch` (server branch) e no proxy BFF (`/api/proxy/[...path]/route.ts`), com fallback `?? NEXT_PUBLIC_API_URL` para dev fora do Docker.
- `middleware.ts`: `/api/*` (antes só `/api/auth`) agora bypassa o redirect de sessão — o proxy faz seu próprio auth check.
- Teste novo em `client.test.ts` cobrindo a precedência `API_INTERNAL_URL`.
