# Changelog — 2026-06-08

## fix(auth): resolver fluxo completo de autenticação em Docker

### Problemas corrigidos

**`api/auth/error?error=Configuration`**
- `apps/web/.env.local` inexistente → Next.js não encontrava `AUTH_SECRET`
- Incompatibilidade NextAuth v5: `Credentials` provider requer `session: { strategy: "jwt" }` — database sessions não suportadas com Credentials

**Google OAuth — AdapterError**
- `PrismaAdapter.createUser` tentava gravar campo `image` ausente no schema
- Adicionado `image String?` ao modelo `User` + migration `20260608000000_add_user_image`
- Aplicado diretamente no banco Docker via `ALTER TABLE`

**Prisma engine ausente no container**
- `next build --output standalone` não inclui binários `.node` automaticamente
- Corrigido via `outputFileTracingIncludes` em `next.config.ts` com glob `libquery_engine-*.node`

**Web service sem acesso ao banco**
- Serviço `web` no docker-compose estava na rede `frontend` apenas; sem `DATABASE_URL`
- Server actions e callbacks do auth usam Prisma diretamente → adicionada rede `backend` + `DATABASE_URL` + `depends_on: db`

**Gmail SMTP 535**
- App Password revogado → atualizado `GMAIL_APP_PASSWORD` no `.env`
- `email.ts` migrado de `service:"gmail"` para config explícita STARTTLS (`host`/`port 587`/`secure: false`)
- Adicionados `SMTP_HOST` e `SMTP_PORT` ao `.env` e ao docker-compose

**OOM no build Docker**
- `next build` excedia heap durante compilação
- Corrigido: `NODE_OPTIONS=--max_old_space_size=4096` no Dockerfile

### Mudanças de comportamento

- **Fluxo de cadastro simplificado**: `requestSignupOtp` → `requestSignup`; cria usuário sem enviar email; redireciona para `/login?registered=1` com banner "Conta criada com sucesso!"
- **Tratamento de erros SMTP**: `requestLoginOtp` e `resendOtp` capturam falhas de email e retornam mensagem de erro em vez de 500
